# Кейс: Тестирование формы регистрации (staging.sortula.ru)

**Что сделано:** декомпозиция требований, выявление серых зон, тест-анализ с применением классов эквивалентности и граничных значений, чек-лист из 25 пунктов, сквозное ручное тестирование формы регистрации (email-флоу и Telegram-флоу), включая проверку через DevTools Network.

**Найдено дефектов:** 7, из них Critical: 1, Major: 3, Minor: 3.

**Ключевые находки:**
- Критический разрыв в пользовательском флоу авторизации через Telegram (сервис технически работает, но пользователь не может пройти его без посторонней подсказки)
- Проблема с недоставкой писем-подтверждений, потенциально блокирующая завершение регистрации
- Систематическая ошибка в сообщениях валидации для кириллических email (подтверждена на двух примерах)
- Утечка технического ответа API вместо человекочитаемого сообщения об ошибке

**Инструменты:** ручное тестирование, Chrome DevTools (Network tab) для анализа запросов и статус-кодов, декомпозиция входных данных по классам эквивалентности и граничным значениям.

**Методология:** каждая находка проверена на воспроизводимость минимум дважды перед фиксацией как баг (например, гипотеза про символ «+» в email была опровергнута дополнительным тестом с точкой, что привело к более точному пониманию проблемы через уточнение у команды).
