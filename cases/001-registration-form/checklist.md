## Поле «Эл. почта»

- [x] Принимает валидный email (например, test@example.com)
- [x] Отклоняет email без символа @
- [x] Отклоняет email без домена (например, test@)
- [x] Отклоняет пустое поле при попытке отправки формы
- [x] Поведение при вводе кириллического email (тест@пример.рф) → БАГ: некорректный текст ошибки, см. BUG-01
- [x] Поведение при вводе email с пробелом в начале/конце — пробелы автоматически обрезаны, регистрация прошла успешно (ожидаемое поведение)
- [x] Поведение при очень длинном email (100+ символов) — граничное значение → БАГ: отсутствует валидация длины, см. BUG-03
- [x] Поведение при email, который уже зарегистрирован в системе — корректно показывает сообщение о том, что email уже занят

## Чекбокс «Согласен с договором-офертой»

- [x] Кнопка «Зарегистрироваться» неактивна без согласия — корректно неактивна, показывает подсказку «сначала согласитесь»
- [x] Ссылка «договором-офертой» (на форме регистрации) открывается и ведёт на реальный документ
- [x] Открытие ссылки «договором-офертой» (на форме регистрации) происходит в новой вкладке (не теряется заполненная форма)
- [x] Ссылка «Договор-Оферта» (в подвале страницы) открывается и ведёт на тот же документ → БАГ: открывается в текущей вкладке, а не в новой, см. BUG-05

## Кнопка «Зарегистрироваться»

- [x] Активна/неактивна в зависимости от заполненности формы
- [x] При успешной отправке — понятная обратная связь пользователю (сообщение, редирект, письмо на почту) → ЧАСТИЧНО: сообщение на экране корректно и мгновенно; письмо на почту не приходит (проверено на 4 адресах, 2 домена), см. BUG-02
- [x] Поведение при повторном клике (защита от дублирования запроса) — форма/кнопка мгновенно скрывается после первого клика, повторный клик физически невозможен; подтверждено в Network (только один запрос `register`, статус 201)

## Регистрация через Телеграм

- [x] Кнопка «Регистрация через Телеграм» открывает ожидаемый флоу
- [x] Флоу завершается успешным созданием аккаунта → БАГИ: кнопка «START BOT» не инициирует /start автоматически (BUG-06); ссылка авторизации не сопровождается инструкцией для пользователя (BUG-07)

## Переход «Уже есть аккаунт? Вход»

- [x] Не теряются данные, если пользователь передумал и вернулся назад
- [x] Форма логина: реакция на невалидный пароль → БАГ: сырой JSON вместо читаемого сообщения, см. BUG-04
